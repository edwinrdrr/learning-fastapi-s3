# Glossary

Plain-language definitions of every term in this course. Skim it now; come back
whenever a word is fuzzy.

### Core
- **API** — a contract that lets programs talk to each other over the web:
  request in, response out.
- **Client** — the side that *asks* (browser, app, script, you with `curl`).
- **Server** — the side that *answers* (our FastAPI app).
- **Endpoint / route** — one URL + method your API responds to, backed by one
  function. E.g. `GET /scrape/{day}`.
- **Resource** — a "thing" the API exposes, named by a URL (a day of scrape data).

### HTTP
- **HTTP** — the rules clients and servers use to exchange requests/responses.
- **Method / verb** — the action: `GET` (read), `POST` (create), `PUT`/`PATCH`
  (update), `DELETE` (remove).
- **URL / path** — the address naming a resource: `/scrape/2026-03-15`.
- **Query parameter** — options after `?`: `?page=1&page_size=100`.
- **Header** — metadata on a request/response (auth keys, content type).
- **Body / payload** — the data carried in a request or response (usually JSON).
- **Status code** — a 3-digit result: **2xx** success, **4xx** client error,
  **5xx** server error. (200, 404, 422, 405, 401, 429, 503 in this project.)
- **REST** — a popular *style* for designing web APIs (name things with URLs, use
  GET to read, etc.). Our API is roughly RESTful.

### Data
- **JSON** — a text format for structured data: objects `{}`, arrays `[]`,
  strings, numbers, booleans, null. The API's input/output format.
- **Serialization** — turning in-memory data into a format to send/store (e.g.
  Python objects → JSON text). **Deserialization** is the reverse.
- **Schema** — a description of data's shape (fields + types).
- **Pydantic** — the Python library FastAPI uses to define schemas and **validate**
  data automatically.
- **Validation** — checking incoming data is well-formed; invalid → `422`.

### FastAPI / framework
- **FastAPI** — the Python library for building the API.
- **Decorator** — the `@router.get("/path")` line that attaches a function to a
  route.
- **Router** — a group of related endpoints (`app/routers/scrape.py`).
- **Dependency / `Depends`** — a function run before your endpoint to provide a
  value or enforce a rule (validation, auth). The pattern is **dependency
  injection**.
- **`response_model`** — declares the shape of what an endpoint returns.
- **Middleware** — code that wraps *every* request/response (CORS, rate limiting).
- **OpenAPI** — a machine-readable description of the whole API, auto-generated by
  FastAPI. **Swagger UI** (`/docs`) and **ReDoc** (`/redoc`) are web viewers for
  it; `/openapi.json` is the raw spec.
- **ASGI / uvicorn** — ASGI is the interface between the web server and FastAPI;
  **uvicorn** is the server program that actually runs your app.

### Storage / data engineering
- **S3** — Amazon's object storage (files in buckets). **MinIO** is an
  S3-compatible server we run locally to stand in for it.
- **Bucket** — a top-level container for objects (ours: `readings`).
- **Object / key** — a stored file and its name; the key looks like a path but
  isn't a real folder structure.
- **Partitioning** — organizing files by a value (here, date: `dt=2026-03-15/`) so
  a query reads only the relevant files (**partition pruning**).
- **Parquet** — a compressed, columnar file format; fast to read slices of. Used
  internally for storage (vs JSON for the API).
- **DuckDB** — a small embedded database that queries Parquet files with SQL,
  used to read + serialize data quickly.
- **Caching** — keeping a local copy of fetched data so you don't re-fetch it.
- **Streaming** — sending a response in chunks instead of all at once (flat
  memory, faster start).
- **Pagination** — returning data in pages (`page` + `page_size`).

### Production
- **Authentication** — verifying *who* is calling (here: an API key header).
- **CORS** — rules for which web origins a browser may let call your API.
- **Rate limiting** — capping how many requests a caller can make (→ `429`).
- **Docker / container** — a packaged, runnable copy of the app + its
  dependencies. **docker-compose** runs several containers together.
- **Environment variable** — a config value supplied from outside the code.
- **IAM role** — AWS's way to give an app temporary credentials without static
  keys.
- **Reverse proxy** — a server (Nginx/Traefik) in front of your app for TLS,
  routing, etc.
